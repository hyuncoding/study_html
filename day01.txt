---- UI, UX
	UI (USER INTERFACE)
		- 사용자가 보는 화면
		- ex) 입력창, 이미지, 폰트, 색상, 글, 그림, 버튼..
	UX (USER EXPERIENCE)
		- 사용자 경험
		- 사용자가 UI를 이용함에 있어 느낄 수 있는 만족감
		- ex) 사용자가 사이트를 이용하면서 불편함이 없는가?

	=> UX가 고려되어있지 않다는 것은 
	      사용자가 UI를 이용함에 있어 불편함이나 제약 사항이 있다는 것.
--------
	프론트엔드
	: UI, UX(사용자가 보는 화면) - 퍼블리싱 (퍼블리셔)
	  사용자의 행위에 따라 API(백엔드)를 통해 데이터를 요청(request)
	  서버로부터 응답받은(response) 데이터를 사용자에게 보여주는 역할

	백엔드
	: 사용자(client)가 요청한 데이터를 DB에 연결하여 요청 의도에 맞는 데이터를
	처리한 후 가공하여 사용자에게 전달

	* BFF
--------
	HTTP(Hyper Text Transfer Protocol)
	(https://www.naver.com)

	인터넷 상에서 서버끼리 데이터를 주고받을 수 있도록 사용하는 규칙의 일종
	s여부는 보안적 요소 추가

	url 주소에 http를 붙이면 해당 규약을 통해 request와 response를 주고
	받겠다. HTTP REQUEST MESSAGE, HTTP RESPONSE MESSAGE
